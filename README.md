# 🎬 Roleta de Filmes — IMDb Top 250

Aplicação web para um casal decidir, em segundos, qual filme assistir — sorteando aleatoriamente entre os 250 melhores filmes do ranking do IMDb, com controle de "já assistidos" persistido no Supabase.

## Stack

- [Next.js 16](https://nextjs.org) (App Router) + [React 19](https://react.dev) + TypeScript
- [Tailwind CSS v4](https://tailwindcss.com) (tema configurado via CSS em `app/globals.css`, sem `tailwind.config`)
- [shadcn/ui](https://ui.shadcn.com) (button, card, switch, dialog, alert-dialog, badge, skeleton, sonner)
- [Framer Motion](https://motion.dev) para as animações (roleta, revelação, confete)
- [Supabase](https://supabase.com) (`@supabase/supabase-js`) para persistir os filmes assistidos

## Estrutura do projeto

```
/app                     # rotas (App Router): layout, page, tema global
/components
  /roulette               # RouletteApp (orquestrador) e todas as peças da roleta
  /layout                 # Header, WatchedToggle

/hooks                    # useMovieDraw (state machine do sorteio), useWatchedMovies (Supabase)
/lib
  /supabase/client.ts      # client Supabase (lazy, lança erro amigável sem env vars)
  utils.ts                  # helper cn() do shadcn
/services
  moviesService.ts          # leitura do dataset local (/data/top250.json)
  watchedService.ts          # CRUD na tabela filmes_imdb do Supabase
/types                     # Movie, WatchedRow
/utils                     # random.ts, format.ts
/data/top250.json           # dataset estático com os 250 filmes
```

## Dados dos 250 filmes

O IMDb não oferece uma API pública oficial para o Top 250, então os dados (posição, título, ano, duração, gêneros, nota, pôster, sinopse, direção, elenco, prêmios) foram consolidados em `data/top250.json`, um snapshot público de **2022-04-03** com a ordem exata do ranking, incluindo os links dos pôsteres (hospedados no CDN do próprio IMDb, `m.media-amazon.com`).

Como o ranking do IMDb muda com o tempo, o arquivo pode ficar levemente desatualizado. Para atualizá-lo, basta substituir `data/top250.json` por um novo dataset no mesmo formato (veja `types/movie.ts`) — nenhuma outra parte do código precisa mudar.

## Configurando o Supabase

A tabela já utilizada pelo app (`filmes_imdb`) segue este schema:

```sql
create table public.filmes_imdb (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  nome_filme text null,
  assistido boolean null,
  ranking_imdb integer null,
  constraint filmes_imdb_pkey primary key (id)
);
```

1. Crie um projeto gratuito em [supabase.com](https://supabase.com) (se ainda não tiver um).
2. Garanta que a tabela `filmes_imdb` acima existe (SQL Editor do Supabase).
3. Como a tabela não tem uma `unique constraint` em `ranking_imdb`, o app faz "select antes de gravar" (atualiza se já existir um registro com aquele ranking, senão insere) em vez de usar upsert nativo. Se quiser reforçar a integridade dos dados, rode opcionalmente:
   ```sql
   create unique index if not exists filmes_imdb_ranking_imdb_key
     on public.filmes_imdb (ranking_imdb);
   ```
4. Se a tabela tiver **Row Level Security (RLS)** habilitada (padrão do Supabase para tabelas criadas pelo painel), crie policies permitindo leitura/escrita com a chave `anon` — como este é um app pessoal sem autenticação, uma policy permissiva é suficiente:
   ```sql
   alter table public.filmes_imdb enable row level security;

   create policy "Permitir leitura" on public.filmes_imdb
     for select to anon using (true);

   create policy "Permitir escrita" on public.filmes_imdb
     for insert to anon with check (true);

   create policy "Permitir atualização" on public.filmes_imdb
     for update to anon using (true) with check (true);
   ```
5. Em **Project Settings > API**, copie a **Project URL** e a **anon public key**.
6. Copie `.env.local.example` para `.env.local` e preencha:
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://SEU-PROJETO.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=sua-chave-anon-public
   ```

> A interface funciona normalmente (sorteio, animação, revelação) mesmo sem `.env.local` configurado — apenas as ações que gravam no Supabase ("Marcar como já visto" / "Vamos assistir" / "Reiniciar histórico") mostram um aviso (toast) explicando o que falta.


## Como funciona

- **Sortear Filme**: gira uma animação de números (1–250) por 3–5s e revela o filme correspondente à posição sorteada.
- **🎲 Sortear novamente**: repete o sorteio dentro do conjunto elegível atual.
- **👁 Marcar como já visto**: grava `assistido = true` no Supabase sem sair da tela.
- **🍿 Vamos assistir**: ação principal — dispara confete + mensagem comemorativa e grava `nome_filme`, `ranking_imdb`, `assistido = true`.
- **📄 Ver detalhes**: abre um modal com sinopse, direção, elenco e prêmios.
- **Toggle "Incluir filmes já vistos"** (canto superior direito): quando desligado, remove do sorteio todos os filmes já marcados como assistidos. Se todos os 250 já tiverem sido assistidos, a tela mostra uma mensagem de parabéns com a opção de reiniciar o histórico.

## Nota sobre o ambiente de desenvolvimento

Este projeto foi criado dentro de uma pasta sincronizada pelo iCloud Drive. Isso funciona, mas o iCloud tenta sincronizar continuamente a pasta `node_modules` (milhares de arquivos pequenos), o que pode deixar o Finder/iCloud lento. Se notar lentidão, considere:

- Mover o projeto para fora do iCloud Drive (ex.: `~/Developer/roleta-filmes`), ou
- Adicionar `node_modules` à lista de itens não sincronizados do iCloud.

Além disso, o Node.js não estava instalado neste Mac (e o Homebrew local está com um bug de detecção da versão do macOS). O ambiente foi preparado baixando o Node.js LTS oficial (v24) diretamente de nodejs.org e criando links simbólicos em `/opt/homebrew/bin` para disponibilizar `node`/`npm`/`npx` no terminal — nenhuma alteração de sistema além disso foi feita. Para uma instalação definitiva e gerenciável (ex.: para trocar de versão do Node no futuro), recomenda-se instalar via [nvm](https://github.com/nvm-sh/nvm) ou corrigir o Homebrew.
